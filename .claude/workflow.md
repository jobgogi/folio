# 개발 워크플로우 (Harness Engineering)

## 원칙
AI가 생성한 코드를 테스트 하네스로 검증한다.
테스트를 통과한 코드만 살아남는다.

## TDD 사이클
1. 테스트 작성 → Red 확인 (필수)
2. 최소 구현 → Green
3. 리팩터링 → 하네스 재실행

## Claude Code 지시 방법

### 좋은 지시 ✅
다음 순서로 진행해:
1. 현재 테스트 상태 확인
2. {기능}에 대한 테스트 먼저 작성 (Red 확인)
3. 테스트를 통과하는 최소 구현 (Green)
4. 전체 테스트 통과 확인
5. 통과하면 커밋

### 나쁜 지시 ❌
{기능} 만들어줘
